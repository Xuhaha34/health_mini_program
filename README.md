# 个人健康管理系统 - 项目总览

## 项目简介

基于前后端分离架构的个人健康管理系统，包含 **微信小程序**（用户端）、**Vue3 Web 管理后台**（管理员端）和 **Django RESTful API 服务端** 三个子项目。系统提供健康记录（体征/饮食/运动）、健康计划与打卡、健康资讯发布阅读、AI 智能健康问答等完整功能。

## 技术栈总览

| 层级 | 技术选型 | 说明 |
| --- | --- | --- |
| **小程序端** | 微信小程序原生框架 + WXML/WXSS/JS + ECharts | 面向普通用户的健康管理客户端 |

| **认证方式** | JWT (JSON Web Token) | 无状态 Token 鉴权 |

## 功能模块详解

### 1. 用户系统

- **注册/登录**：用户名 + 密码，JWT Token 认证
- **个人信息**：昵称、性别、年龄、身高、体重、邮箱
- **密码修改**：支持修改登录密码
- **权限体系**：普通用户（只能操作自己的数据）/ 管理员（可访问平台统计）

### 2. 健康记录系统

| 记录类型 | 数据字段 |
| --- | --- |
| **体征数据** | 日期、体重、收缩压、舒张压、心率、血糖、备注 |
| **饮食记录** | 日期、餐别、食物名称、热量、蛋白质、脂肪、碳水化合物 |
| **运动记录** | 日期、运动类型、运动时长、消耗热量、备注 |

全部支持增删改查操作，数据与用户账户关联。

### 3. 健康计划系统

- **计划类型**：饮食计划、运动计划、体征管理、综合计划
- **计划属性**：标题、目标值、开始日期、结束日期、状态（进行中/已完成/已放弃）、是否提醒
- **每日打卡**：每个计划每天可打卡一次，记录完成情况
- **进度追踪**：自动统计打卡天数、连续打卡天数、总打卡次数

### 4. 健康资讯系统

- **资讯分类**：支持多级分类管理
- **内容发布**：后台发布图文资讯
- **上下线控制**：管理员可控制资讯显示状态
- **阅读统计**：记录浏览量

### 5. AI 健康问答系统（核心亮点）

- **智能对话**：用户输入健康问题，AI 给出专业建议
- **上下文记忆**：每次请求携带最近 6 条对话历史，保持对话连贯性
- **个性化回答**：AI 结合用户的年龄/性别/体重/血压/血糖/健康计划数等信息
- **多供应商支持**：统一接入 OpenAI/DeepSeek/智谱 AI/通义千问/豆包 等主流大模型
- **自动降级**：外部模型不可用或未配置 API Key 时，自动回落到本地关键词知识库（覆盖 20+ 健康主题），保证功能始终可用
- **对话记录**：自动保存所有用户对话历史到数据库

## API 接口规范

### 统一响应格式

所有接口返回统一的 JSON 格式：

```json
{
    "code": 200,
    "message": "操作成功",
    "data": {}
}
```

### 常用状态码

| code | 说明 |
| --- | --- |
| 200 | 操作成功 |
| 400 | 参数错误 |
| 401 | 未登录 / Token 无效 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 409 | 冲突操作（如重复打卡） |
| 500 | 服务端异常 |

### 接口总览

| 模块 | 路由前缀 | 主要功能 |
| --- | --- | --- |
| 用户 | `/api/users/` | 注册、登录、个人信息、修改密码 |
| 健康记录 | `/api/records/` | 体征、饮食、运动数据 CRUD |
| 健康计划 | `/api/plans/` | 计划管理、打卡接口 |
| 健康资讯 | `/api/news/` | 资讯列表、详情、分类 |
| AI 问答 | `/api/ai/` | 智能对话、AI 配置查询 |
| 平台统计 | `/api/statistics/` | 数据看板接口（仅管理员） |

## 快速开始（三端联动）

### 前置要求

| 软件 | 版本要求 |
| --- | --- |
| Python | 3.10 及以上 |
| MySQL | 8.0 及以上 |
| Node.js | 16 及以上 |
| 微信开发者工具 | 最新稳定版 |

---

### 第一步：启动 Django 后端

```powershell
# 进入后端目录
cd health_manage_backend

# 安装 Python 依赖
pip install -r requirements.txt

# 配置数据库（编辑 settings.py 中的 DATABASES）
# 确保 MySQL 服务已启动，数据库 health_manage_db 已创建

# 初始化数据库
py manage.py makemigrations
py manage.py migrate

# 创建管理员账号（管理后台登录用）
py manage.py createsuperuser

# 启动开发服务器
py manage.py runserver 0.0.0.0:8000
```

后端 API 地址：`http://127.0.0.1:8000/api/`

---

### 第二步：启动 Vue 管理后台（可选，用于管理员）

```powershell
# 新开一个终端窗口，进入前端目录
cd health_admin_web

# 安装 npm 依赖
npm install

# 配置后端 API 地址（编辑 .env 文件）
# VITE_API_BASE_URL=http://127.0.0.1:8000/api

# 启动开发服务器
npm run dev
```

管理后台地址：`http://localhost:5173`

登录账号：使用第一步中 `createsuperuser` 创建的账号

---

### 第三步：运行微信小程序

1. 打开「微信开发者工具」
2. 导入项目：选择 `微信小程序代码/` 目录
3. AppID 选择「测试号」（或填写自己的正式 AppID）
4. 修改 `app.js` 中的 `baseUrl` 为 `http://127.0.0.1:8000/api`
5. 点击「编译」运行小程序
6. 注册账号后开始使用！

**提示**：微信开发者工具 → 详情 → 本地设置 → 勾选「不校验合法域名...」以支持本地 HTTP 请求

## AI 大模型配置（可选但推荐）

### 配置方式

**方式 A：修改 Django settings.py**

编辑 `health_manage_backend/health_manage/settings.py`，在文件末尾添加：

```python
# AI 大模型配置
AI_PROVIDER = "zhipu"          # 可选: local / openai / deepseek / zhipu / qwen / doubao
AI_API_KEY = "你的-API-KEY"
AI_MODEL = ""                  # 留空使用供应商默认模型
AI_TEMPERATURE = 0.7
AI_MAX_TOKENS = 1000
AI_TIMEOUT = 30
```

修改后重启 Django 服务即可。

**方式 B：使用环境变量（推荐，避免密钥泄露）**

```powershell
# Windows PowerShell
$env:AI_PROVIDER = "deepseek"
$env:AI_API_KEY = "你的-API-KEY"
py manage.py runserver 0.0.0.0:8000
```

## 常见问题 FAQ

### Q1: 启动 Django 时报 ModuleNotFoundError

**A**: 确认已在 `health_manage_backend` 目录执行 `pip install -r requirements.txt`

### Q2: 小程序提示「网络请求失败」

**A**: 请检查：
1. Django 后端是否启动（`http://127.0.0.1:8000`）
2. `app.js` 中的 `baseUrl` 是否正确
3. 微信开发者工具中是否勾选了「不校验合法域名」

### Q3: Vue 管理后台登录提示 401

**A**: 请确认已通过 `py manage.py createsuperuser` 创建了管理员账号，且后端服务正常运行。

### Q4: AI 问答总是返回本地知识库答案

**A**: 这是预期行为（零配置默认方案）。如需接入外部大模型，请参考上方「AI 大模型配置」章节，在 `settings.py` 中配置 `AI_PROVIDER` 和 `AI_API_KEY`。

### Q5: 健康计划打卡返回「计划不存在」

**A**: 普通用户只能打卡自己创建的计划。请确认当前登录用户创建了该计划，或使用管理员账号操作。

### Q6: 数据库连接失败

**A**: 检查 `settings.py` 中的数据库配置，确认 MySQL 服务启动，数据库 `health_manage_db` 已创建，用户名密码正确。
